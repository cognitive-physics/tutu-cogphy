# 认知翻译插件 · 编码器：贝叶斯-IFD 推断引擎

> 从天然读人机制中抽取出的第一层：把模糊的人类信号翻译成结构化参数估计。

## 功能定位

三层架构中的**第一层**：

1. **编码器**：把模糊的人类信号翻译成结构化的 `ρ` 场，降低人机之间的 `D_KL`
2. 四维决策引擎：`δA = 0` 实时求解
3. 输出解码器：把最优路径翻译回自然语言

编码器的核心流程：

**接收原始输入 → 多通道采样 → 逆问题求解 → 贝叶斯更新 → 噪声过滤 → 输出结构化参数估计**

## 五步流程

### Step 1 · 全通道并行采样

同时从多个观测通道采样对方的信息密度分布 `ρ`：

| 通道 | 采样内容 | 对应含义 |
|---|---|---|
| 语言内容 | 说了什么（语义） | `ρ` 场低频分量 |
| 语言结构 | 怎么说（逻辑、跳跃） | `H` 的结构特征 |
| 语气/情绪色彩 | 带什么情绪说 | `σ*` 信号强度 |
| 行为/动作 | 做了什么、怎么做 | `δA = 0` 的实际执行路径 |
| 微表情/神态 | 无意识泄露的信号 | `μ` 的直接观测 |
| 言行一致性 | 说的和做的是否匹配 | `μ` 的交叉验证 |
| 上下文/时间线 | 什么情境下发生 | 边界条件 + 外部 `λ` 场 |

### Step 2 · 逆问题求解

观测到对方的输出 `ρ(t)`，反推对方的内部参数：

```math
ρ(t) \\xrightarrow{\\text{inverse}} (ρ_0, H_0, μ, σ^*, λ_{history})
```

输出不是确定值，而是后验分布：

```math
P(θ \\,|\\, D_{observed}) \\propto P(D_{observed} \\,|\\, θ) \\cdot P(θ)
```

其中 `θ = (ρ_0, H_0, μ, σ^*)`。

### Step 3 · 贝叶斯逐帧更新

每次新观测到来，更新参数估计：

```math
P(θ \\,|\\, D_{1:n}) =
\\frac{P(D_n \\,|\\, θ) \\cdot P(θ \\,|\\, D_{1:n-1})}{P(D_n \\,|\\, D_{1:n-1})}
```

与认知留白 `R` 的关系：

- 数据少时，`R` 大，保持留白，不轻易锁定结论
- 数据多且一致时，`R \\to 0`

### Step 4 · `D_KL` 自动降噪

当不同通道信号互相矛盾时：

```math
D_{KL}(ρ_{A} \\,\\|\\, ρ_{B}) > θ_{noise}
\\Rightarrow \\text{标记不一致并降权}
```

降噪的含义不是删除，而是：

- 不一致信号降低权重
- 同时把“此处可能存在较高 `μ`”作为额外信息保留

### Step 5 · 收敛判定

```math
\\|P(θ \\,|\\, D_{1:n}) - P(θ \\,|\\, D_{1:n-1})\\| < ε
\\quad \\text{持续 } k \\text{ 帧}
```

当新数据不再显著改变估计时，模型收敛，可以锁定结论。

## 工程化方向

1. **通道数字化**：不同场景可采样通道不同
2. **先验库**：需要人格模式库作为贝叶斯先验
3. **更新频率**：可先从“一轮对话更新一次”开始
4. **过拟合防护**：数据不足时强制保持不确定性
